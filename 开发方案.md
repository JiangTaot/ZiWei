## 四、API 接口清单

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/ziwei/chart/calculate` | 排盘计算 | 可选（已登录则关联 userId） |
| GET | `/api/ziwei/chart/{id}` | 获取命盘详情 | 可选（有主命盘校验归属） |
| GET | `/api/ziwei/chart/my-list` | 当前用户的命盘列表 | **强制**（需 JWT） |
| DELETE | `/api/ziwei/chart/{id}` | 删除命盘（校验归属） | **强制**（需 JWT） |
| POST | `/api/ziwei/user/login` | 微信登录 | 否 |
| GET | `/api/ziwei/user/profile` | 用户信息 | 是 |
| POST | `/api/ziwei/ai/interpret/{id}` | AI 解读（同步） | 是 |
| POST | `/api/ziwei/ai/interpret/{id}/stream` | AI 解读（SSE） | 是 |

### 认证机制（V2 更新）

三层认证架构：
1. **OptionalJwtFilter** — 在所有 `/api/**` 请求上先于拦截器执行，有 token 则提取 userId，无 token 则放行
2. **JwtInterceptor** — 白名单路径跳过；`GET /chart/{数字ID}` 可选认证；其余路径强制认证
3. **Controller 层** — `/{id}` 详情和删除接口额外校验命盘归属

白名单：`/user/login`、`/chart/calculate`、Swagger/Knife4j 文档路径

## 七、开发命令

```bash
# 后端
cd ziwei-server
mvn spring-boot:run          # 启动 (端口 48121)

# 前端
cd ziwei-app
npm run build:mp-weixin       # 编译到 dist/build/mp-weixin/
# 微信开发者工具 → 打开 dist/build/mp-weixin/

# 代码质量
mvn clean compile              # 后端编译验证
npx uni build -p mp-weixin     # 前端编译验证
```
